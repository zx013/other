物体(object)
描述一个物体，包括静止的石头、花草树木等，活动的人、动物等


技能(skill)
描述一个技能，属于一种特殊的物体
技能前提（能使用技能的状况，如有些技能必须作用于物体）
影响的范围
移动的轨迹
连接的物体
施加给物体的状态


状态(buffer)
描述一种状态，可以改变物体的属性

由技能施加给物体后，在下一个时间片生效

同种状态可添加到同一个时间片中，实现修改的效果可在状态的实现中完成
(刷新持续时间)


事件(event)

定时器事件
定时器的每个时间点生成一个事件

属性变更事件
物体属性变更时，绑定在变更前后的事件
（伤害加成，回复加成，薄葬等）

状态触发事件
状态生效时，绑定在生效前后的事件
（）

硬件事件
鼠标、键盘等硬件设备产生的事件

操作事件
对物体进行操作的事件，通常由硬件事件判断生成


移动
右键点击 -> 硬件事件 -> 事件分析响应 -> 操作事件 -> 自身事件响应 -> 创建移动技能（连接自身） -> 技能碰撞 -> 将移动状态添加到物体（自身）
时钟事件 -> 移动状态 -> 移动实现（预移动 -> 碰撞检测 -> 调整方向 -> 向目标点移动）

攻击
右键点击 -> 硬件事件 -> 事件分析响应 -> 操作事件 -> 自身事件响应 -> 创建攻击技能（连接目标） -> 技能碰撞（弹道飞行） -> 将攻击状态添加到物体（目标）
时钟事件 -> 攻击状态 -> 攻击实现（获取自身攻击 -> 获取目标防御 -> 计算伤害 -> 获取目标生命 -> 生命扣除）

技能
选择技能
键盘点击 -> 硬件事件 -> 事件分析响应（选择技能）
释放技能
鼠标点击 -> 硬件事件 -> 事件分析响应 -> 操作事件 -> 自身事件响应 -> 创建技能 -> 技能碰撞 -> 将状态添加到目标
时钟事件 -> 技能状态 -> 技能实现

不能作用于目标的情况（点目标）
自身事件响应 -> 获取选择点的目标物体 -> 判断目标物体能否作用技能

事件分析响应获取硬件事件，转换为操作事件
不同设备只需建立不同的事件分析响应，后面的流程保持一致
